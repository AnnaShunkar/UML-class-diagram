# Опис предметної області (фільми)
Система бронювання квитків для кінотеатрів дозволяє користувачам переглядати розклад фільмів, обирати сеанс та конкретні місця у залі. Кожен квиток прив’язаний до певного фільму, часу показу та місця. Користувачі можуть реєструватися, здійснювати бронювання та переглядати історію своїх відвідувань. Адміністратори додають нові фільми, керують розкладом та доступністю місць.
# Архітектура 
У цій системі бронювання квитків на фільми використано клієнт‑серверну багатошарову архітектуру:
- Клієнтський шар (Frontend) — інтерфейс для користувача (HTML/React), який відправляє HTTP‑запити до сервера.
- Серверний шар (Backend) — застосунок (Flask або Spring Boot), що реалізує бізнес‑логіку: реєстрація користувачів, бронювання місць, робота з розкладом фільмів.
- Шар даних (Database) — PostgreSQL, де зберігаються користувачі, фільми, сеанси, місця та бронювання.
- Кешуючий шар (Redis) — використовується для швидкого доступу до часто запитуваної інформації (наприклад, розклад сеансів або доступність місць), щоб зменшити навантаження на базу даних.
Таким чином, архітектура чітко розділяє UI → API → Data, що відповідає принципам layered architecture у клієнт‑серверній моделі.
